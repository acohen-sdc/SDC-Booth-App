# Claude Instructions ‚Äî SDC Kiosk App

## Project Summary
Single-file React kiosk app for Steven Douglas Corp. trade-show booth (Automate 2026).
All content lives in `SDC Kiosk App.html`. The deployable version is `SDC Kiosk App - Self Contained.html` (generated by `bake_assets.py`).

---

## Key Rules

### Always edit the SOURCE file
- **Source:** `SDC Kiosk App.html` ‚Äî edit this for any content or code changes
- **Built output:** `SDC Kiosk App - Self Contained.html` ‚Äî never edit this by hand; it is regenerated by the build script

### Rebuild after every content or asset change
```bash
pip install Pillow   # first time only
python bake_assets.py
```
This produces a new `SDC Kiosk App - Self Contained.html`. Always commit both files together.

### Never add npm, node_modules, or a bundler
The app intentionally has no build toolchain. React and Babel are loaded via CDN script tags. Keep it that way.

## Adding Content

### New case study
1. Add photo to `Image Files/` (JPG or PNG, any size ‚Äî bake script will compress)
2. Add entry to `caseStudies: [...]` in `SDC_DATA`:
```js
{
  id: "unique-kebab-id",
  title: "Machine Title",
  industry: "Life Sciences",          // must match an industry name exactly
  industryId: "life-sciences",        // must match an industry id exactly
  capability: "Assembly ¬∑ Robotic Integration",
  blurb: "One-paragraph description.",
  technologies: ["Tech 1", "Tech 2"],
  rate: "140 PPM",
  investment: "$500K‚Äì$1M",
  leadTime: "32‚Äì36 weeks",
  keyImpact: "Key outcome sentence.",
  delivered: "On-Time",
  image: "Exact Filename.jpg",        // must match file in Image Files/
  video: "Exact Filename.mp4"         // must match file in Video Files/, or null
}
```
3. If there's a video, add to `videos: [...]`:
```js
{
  id: "v-unique-id",
  caseId: "unique-kebab-id",          // matches case study id above
  title: "Machine Title",
  industry: "Life Sciences",
  category: "Life Sciences",
  tags: ["Tag 1", "Tag 2", "Tag 3"],
  file: "Exact Filename.mp4"
}
```
4. Add the case id to the matching industry's `cases: [...]` array
5. Run `python bake_assets.py`

### New industry icon or capability icon
Drop the SVG into `Icon Files/` and reference it by filename in `SDC_DATA`.

### New customer logo
Drop the PNG into `Customer Logo Files/` and add the filename to `customerLogos: [...]`.
Use transparent-background PNGs only ‚Äî they are displayed white-filtered on a dark panel.

## Routing
The app uses a simple stack ‚Äî no router library:
```js
navigate({ name: "case-study-detail", id: "ignition-coil" })  // push
back()   // pop
home()   // reset to [{ name: "home" }]
```
Valid route names: `home`, `case-studies`, `case-study-detail`, `industries`,
`industry-detail`, `capabilities`, `capability-detail`, `videos`, `video-detail`,
`numbers`, `contact`

### First-time kiosk setup (do once)
- [ ] Copied the full repo to the kiosk machine (USB or git clone)
- [ ] Ran `SETUP - Create Desktop Shortcut.ps1` ‚Üí creates desktop launcher
- [ ] Ran `SETUP - Enable Auto Updates.ps1` ‚Üí registers 5-min GitHub sync task
- [ ] Tested launch via `SDC Kiosk` desktop shortcut

### How auto-update works
1. Ashley edits `SDC Kiosk App.html` and runs `python bake_assets.py`
2. She commits and pushes both HTML files to GitHub
3. The kiosk checks GitHub every 5 minutes via `AUTO-UPDATE.ps1`
4. If the self-contained HTML changed, Chrome closes and relaunches with the new version
5. Update log is written to `update-log.txt` on the kiosk (not committed to git)

## What NOT to Do
- Do not edit `SDC Kiosk App - Self Contained.html` directly
- Do not add `node_modules`, `package.json`, or any bundler
- Do not remove the `Video Files/` path fallback in the asset helpers ‚Äî it is the only way videos load
- Do not change the 1920√ó1080 canvas size without also updating the scale-fit JS
- Do not commit the `Video Files/` folder ‚Äî it is gitignored (5‚Äì6 GB)
